# AI-Driven Development (Vibe Coding) Workflow Standard

This document establishes a universal, phase-by-phased development lifecycle and strict coding guidelines for projects built primarily by AI coding agents. 
The goal of this standard is to ensure that human developers never go "blind" to their codebase. Code generated by AI must prioritize human readability, strict modularity, and explicit documentation so a human can instantly debug or take over at any moment.

---

## 🤖 Core Directives for AI Agents

1. **The "No Magic Folder" Rule**
    - Code must be aggressively split into hyper-specific files. 
    - Massive monolithic files (e.g., `index.ts` dumping ground) are strictly forbidden. 
    - Vague filenames like `utils.ts`, `helpers.ts`, or `misc.ts` are forbidden. Name exactly what the file does (e.g., `parse-webhook-payload.ts`).
    - **Exception (`shared-kernel/`):** A single directory under `/src/` is permitted for pure, stateless functions (e.g., date formatters) to avoid extreme DRY violations. It must contain zero business logic or external state.
2. **Mandatory File Map (`CODE_MAP.md`)**
    - The repository WILL contain a living `CODE_MAP.md` file. 
    - Every time an AI agent creates, moves, or deletes a file, it **must** update this map with a 1-sentence summary of the file's purpose. 
    - The `CODE_MAP.md` must be able to answer "where is the code for X?" in 5 seconds.
    - **Domain Grouping:** As the project scales, `CODE_MAP.md` MUST be strictly grouped by **Domain** (e.g., `## Domain: Tenant Processing`) exactly matching the folder structure to prevent AI context degradation.
3. **Intent-Based Comments (The "Why")**
    - Code must be heavily commented, but comments must explain the *human business intent*, not the literal code execution.
    - *Bad:* `// Loop through array and check if balance > 0`
    - *Good:* `// Stripe Billing: Block users who have exhausted their free trial credits.`
4. **Structurally Predictable, "Dumb" Code**
    - Prioritize extreme readability over hyper-condensed syntax. Code must be structured so a human can instantly parse the logical flow.
    - **Self-Documenting Structure:** Use descriptive variable names (`hasExceededTrialBalance` instead of `x`), and avoid deep nesting (e.g., use early returns to keep logic flat).
    - Do not "show off" with complex one-liners or chained ternary operators. If it takes a human more than 10 seconds to decipher a function, it is too complex. Break it down into explicitly named helper functions.
5. **Architectural Purity & Isolation**
    - Distinct business logic domains must **never** be mixed in the same file. 
    - If a monolithic project serves multiple distinct personas or models, their logic must exist in completely isolated directories, unified only at the top-level router.
6. **Test-Driven AI (TDAI)**
    - AI agents must write test assertions logic (e.g., unit tests) *before* or *alongside* writing the business logic. 
    - If a feature works but there is no automated test proving it works, the feature is considered incomplete.

## 🏗️ The 7-Stage AI Development Lifecycle

AI agents must complete these stages sequentially. **No code shall be written for a subsequent stage until the human user explicitly approves the deliverables of the current stage.**

### Stage 1: Architecture & Local Project Structure
- **Goal:** Define the system topography, data flow, external dependencies, and highly discoverable local folder structure before touching code.
- **Guideline: The "Super Fast Search" Rule**
  - Folder trees should be grouped by **Domain / Feature**, not by technical layer (e.g., prefer putting all billing code in `/billing` rather than splitting it between generic `/controllers` and `/models` folders).
- **AI Deliverables:**
  - Create/Update `architecture.md` detailing the system design, routing logic, infrastructure components, AND the explicit file tree map.
  - Detail any 3rd-party APIs, SDKs, or external services required.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 2: Database & State Design
- **Goal:** Design the data layer that will support the application.
- **AI Deliverables:**
  - Create `database_schema.md` with Entity-Relationship tables for all required domains.
  - Document caching strategies or memory state dependencies.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 3: Interface & API Contract Design
- **Goal:** Define exactly what data structures will be passed between the client and server before writing the router or UI logic.
- **AI Deliverables:**
  - Document the REST/GraphQL endpoints or server action signatures.
  - Define TypeScript interfaces or Zod schemas for the expected payloads and responses.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 4: Core Routing & Foundation Implementation
- **Goal:** Scaffold the repository and write the "brain/router" of the application.
- **AI Deliverables:**
  - Initialize the project and install strict dependencies.
  - Initialize `CODE_MAP.md`.
  - Implement the core routing layer or entry point.
  - Write unit tests proving the router correctly handles basic input paths.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 5: Domain-Specific Logic Integrations
- **Goal:** Build out the isolated business models and external tool connections.
- **Client-Side/UI Rule:** All React/UI components must clearly state which backend route/action they communicate with in their code comments. UI components must remain "dumb" display layers.
- **AI Deliverables:**
  - Implement specific modules following the "No Magic Folder" rule.
  - Update `CODE_MAP.md` for every new file.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 6: Security, Billing, & Edge Cases
- **Goal:** Lock down the application and implement operational requirements.
- **AI Deliverables:**
  - Implement authentication, rate limiting, or token metering.
  - Handle payment webhooks and balance deductions.
- **Approval Gate:** ⬜ Waiting for Human Approval

### Stage 7: Deployment & CI/CD
- **Goal:** Move the code from the local environment to the production host.
- **AI Deliverables:**
  - Bundle/compile the application.
  - Provide secure deployment scripts or instructions for the target infrastructure.
- **Approval Gate:** ⬜ Waiting for Human Approval
